3. MİMARİ YAPI
3.1. Yüksek Seviye Mimari
Sistem üç ana bileşenden oluşuyor:
1. Web Frontend (Next.js + Phaser): Kullanıcının gördüğü her şey
2. Backend API (NestJS): Oyun mantığı, veri doğrulama
3. Real-time Server (Socket.io): PvP, anlık güncellemeler
3.2. Server-Authoritative Yaklaşım
Tüm önemli kararlar serverda alınır. Frontend sadece görselleştirir. Bu cheat&apos;i önler:
Kaynak miktarları: Sadece serverda tutulur
Birim sağlığı: Server hesaplar
Savaş sonuçları: Server belirler
Tier yükseltme: Server doğrular
Frontend manipülasyonu hiçbir avantaj sağlamaz
3.3. Veri Akışı Örnekleri
Senaryo 1: Birim Üretimi
1. Oyuncu UI&apos;da &apos;Üret&apos; butonuna tıklar
2. Frontend NestJS API&apos;sine istek gönderir
3. Backend kaynak kontrolü yapar (PostgreSQL)
4. Yeterliyse: Kaynaklar düşülür, üretim kuyruğa eklenir
5. Redis&apos;te üretim sayacı başlar (TTL ile)
6. Süre sonunda backend birim oluşturur
7. Socket.io ile oyuncuya &apos;Birim hazır&apos; bildirimi gider
8. Frontend Phaser scene&apos;de birim görselleşir
Senaryo 2: Birleştirme/Mutasyon
1. Oyuncu iki birimi sürükle-bırak ile birleştirir (frontend)
2. Frontend backend&apos;e &apos;merge&apos; isteği gönderir
3. Backend birimlerin uygun olduğunu kontrol eder
4. Eski birimler silinir, yeni Tier birim oluşur
5. Veritabanı güncellenir
6. Frontend güncellenmiş birimi gösterir
Senaryo 3: PvP Savaş
1. İki oyuncu Socket.io &apos;matchmaking&apos; kuyruğuna girer
2. Eşleşen oyuncular özel &apos;battle room&apos; oluşturur
3. Backend savaş simülasyonunu başlatır (saniyede 30 tick)
4. Her tick&apos;te birim hareketleri ve hasar hesaplanır
5. Sonuçlar her iki oyuncuya broadcast edilir
6. Frontend Phaser ile gerçek zamanlı render
7. Savaş sonu: Skor PostgreSQL&apos;e yazılır
